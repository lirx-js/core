# Decision tree

This page contains all the available functions provided by `@lirx/core`.

They are grouped in a decision tree to help you find the right function suitable to your requirements.

Frequently used functions are in **bold**.

### I want to:

#### create an Observable from

*without notifications*

- nothing and emit no value: [empty](/docs/reference/empty/)

- one/single value
  - already defined: **[single](/docs/reference/single/)**
  - defined later: [reference](/docs/reference/reference/)

- a list of values: **[of](/docs/reference/of/)**

- an iterable
  - array: [fromArray](/docs/reference/from-array/)
  - iterable: [fromIterable](/docs/reference/from-iterable/)
  - iterator ⚠️: [fromIterator](/docs/reference/from-iterator/)

- something related to the DOM
  - an EventTarget: **[fromEventTarget](/docs/reference/from-event-target/)**
  - an IntersectionObserver: [fromIntersectionObserver](/docs/reference/from-intersection-observer/)
  - a ResizeObserver: [fromResizeObserver](/docs/reference/from-resize-observer/)
  - a CSS @media query: [fromMatchMedia](/docs/reference/from-match-media/)

- one Observable
  - defined later: [defer](/docs/reference/defer/)

- many Observables. When any value is received
  - re-emit it concurrently: **[merge](/docs/reference/merge/)**
  - combine the values in an array and emit it: **[combineLatest](/docs/reference/combine-latest/)**
    - combine the values in an array, runs a function with these values, and emit distinct returned
      values:
       **[optimizedReactiveFunction](/docs/reference/optimized-reactive-function/)**,
       [reactiveFunction](/docs/reference/reactive-function/)
      - arithmetic:
        [reactiveAdd](/docs/reference/reactive-add/),
        [reactiveAddMany](/docs/reference/reactive-add-many/),
        [reactiveSubtract](/docs/reference/reactive-subtract/),
        [reactiveMultiply](/docs/reference/reactive-multiply/),
        [reactiveDivide](/docs/reference/reactive-divide/)
      - logic:
        [reactiveAnd](/docs/reference/reactive-and/),
        [reactiveAndMany](/docs/reference/reactive-and-many/),
        [reactiveOr](/docs/reference/reactive-or/),
        [reactiveOrMany](/docs/reference/reactive-or-many/),
        [reactiveNot](/docs/reference/reactive-not/)
      - comparison:
        [reactiveEqual](/docs/reference/reactive-equal/),
        [reactiveNotEqual](/docs/reference/reactive-not-equal/),
        [reactiveGreaterThan](/docs/reference/reactive-greater-than/),
        [reactiveGreaterThanOrEqual](/docs/reference/reactive-greater-than-or-equal/),
        [reactiveLowerThan](/docs/reference/reactive-lower-than/),
        [reactiveLowerThanOrEqual](/docs/reference/reactive-lower-than-or-equal/)
      - string:
        [reactiveTemplateString](/docs/reference/reactive-template-string/)

- *time related*
  - emits every 'ms': **[interval](/docs/reference/interval/)**
  - emits after 'ms': [timeout](/docs/reference/timeout/)
  - emits when idle time is available: [idle](/docs/reference/idle/)
  - emits on each animation frame: [fromAnimationFrame](/docs/reference/from-animation-frame/)



*with notifications*

- nothing and send a `complete` Notification: **[emptyWithNotifications](/docs/reference/empty-with-notifications/)**

- one/single value
  - already defined: **[singleWithNotifications](/docs/reference/single-with-notifications/)**

- a list of values: **[ofWithNotifications](/docs/reference/of-with-notifications/)**

- an iterable
  - sync
    - array: [fromArrayWithNotifications](/docs/reference/from-array-with-notifications/)
    - iterable: [fromIterableWithNotifications](/docs/reference/from-iterable-with-notifications/)
    - iterator ⚠️: [fromIteratorWithNotifications](/docs/reference/from-iterator-with-notifications/)

  - async
    - async iterable: [fromAsyncIterable](/docs/reference/from-async-iterable/)
    - async iterator ⚠️: [fromAsyncIterator](/docs/reference/from-async-iterator/)

- something related to the DOM
  - the user's location (Geolocation): [fromGeolocationPosition](/docs/reference/from-geolocation-position/)

- a promise
  - with a factory: **[fromPromiseFactory](/docs/reference/from-promise-factory/)**
  - without a factory ⚠️: [fromPromise](/docs/reference/from-promise/)

- a readable stream
  - [w3c streams](https://streams.spec.whatwg.org/#rs-class)
    - readable stream: [fromReadableStream](/docs/reference/from-readable-stream/)
    - readable stream reader ⚠: [fromReadableStreamReader](/docs/reference/from-readable-stream-reader/)
  - nodejs: TODO

- an http request
  - using fetch: **[fromFetch](/docs/reference/from-fetch/)**
  - [fromFetchJSON](/docs/reference/from-fetch-json/)
  - [fromFetchText](/docs/reference/from-fetch-text/)
  - [fromFetchArrayBuffer](/docs/reference/from-fetch-array-buffer/)
  - [fromFetchBlob](/docs/reference/from-fetch-blob/)
  - [fromFetchStream](/docs/reference/from-fetch-stream/)

[//]: # (  - using xhr: [fromXHR]&#40;src/observable/built-in/from/with-notifications/http/xhr/from-xhr/from-xhr.md&#41;)

- a blob (reads content): [readBlob](/docs/reference/read-blob/)

- an error (send an `error` Notification): **[throwError](/docs/reference/throw-error/)**

- *time related*
  - emits after 'ms':
    - with a complete notification: [timeoutWithCompleteNotification](/docs/reference/timeout-with-complete-notification/)
    - with an error notification: [timeoutWithErrorNotification](/docs/reference/timeout-with-error-notification/)

- many Observables:
  - equivalent of Promise.all: **[forkJoin](/docs/reference/fork-join/)**
  - equivalent of Promise.race: **[raceWithNotifications](/docs/reference/race-with-notifications/)**
  - equivalent of Promise.any: [anyWithNotifications](/docs/reference/any-with-notifications/)

- a `rxjs`'s Observable: [fromRxJS](/docs/reference/from-rxjs/)


#### convert an Observable to

*without notifications*

- a promise: [toPromise](/docs/reference/to-promise/)

*with notifications*

- a promise
  - with only the last value: **[toPromiseLast](/docs/reference/to-promise-last/)**
  - with every value: [toPromiseAll](/docs/reference/to-promise-all/)

- an async iterable: [toAsyncIterable](/docs/reference/to-async-iterable/)

- an `RxJS`' Observable: [toRxJS](/docs/reference/to-rxjs/)


#### create an ObservablePipe which

*without notifications*

- *ObserverPipe related*
  - emits only distinct received values: **[distinctObservablePipe](/docs/reference/distinct-observable-pipe/)**
  - filters received values: **[filterObservablePipe](/docs/reference/filter-observable-pipe/)**
  - transforms received values: **[mapObservablePipe](/docs/reference/map-observable-pipe/)**
  - transforms and filter received values: [mapFilterObservablePipe](/docs/reference/map-filter-observable-pipe/)
  - transforms received values with an accumulator: [scanObservablePipe](/docs/reference/scan-observable-pipe/)
  - reads received values, and re-emits them without transformations: [tapObservablePipe](/docs/reference/tap-observable-pipe/)

- *Source related*
  - allows one Observable to emit its values to many Observable:
    - [shareObservablePipe](/docs/reference/share-observable-pipe/)
    - [shareObservablePipeWithMulticastReplaySource](/docs/reference/share-observable-pipe-with-multicast-replay-source/)
    - **[shareObservablePipeWithMulticastReplayLastSource](/docs/reference/share-observable-pipe-with-multicast-replay-last-source/)**


- reduces the order of an Observable of Observables
  - once: **[mergeAllSingleObservablePipe](/docs/reference/merge-all-single-observable-pipe/)**
  - many: [mergeAllObservablePipe](/docs/reference/merge-all-observable-pipe/)
  - maps an Observable and then reduces the order:
    - once: **[mergeMapSingleObservablePipe/switchMapObservablePipe](/docs/reference/merge-map-single-observable-pipe/)**
    - many: [mergeMapObservablePipe](/docs/reference/merge-map-observable-pipe/)

- accumulates values:
  - until another Observable emits: [bufferObservablePipe](/docs/reference/buffer-observable-pipe/)
  - within a fixed period of time: [bufferTimeObservablePipe](/docs/reference/buffer-time-observable-pipe/)

- take
  - the first value: [firstObservablePipe](/docs/reference/first-observable-pipe/)
  - the X first values: [takeObservablePipe](/docs/reference/take-observable-pipe/)
  - the values until another observable emits: [takeUntilObservablePipe](/docs/reference/take-until-observable-pipe/)
  - the values until a predicate fails: [takeWhileObservablePipe](/docs/reference/take-while-observable-pipe/)
  - the first value that passes a condition: [findObservablePipe](/docs/reference/find-observable-pipe/)

- time related
  - **[debounceTimeObservablePipe](/docs/reference/debounce-time-observable-pipe/)**
  - **[throttleTimeObservablePipe](/docs/reference/throttle-time-observable-pipe/)**
  - [debounceFrameObservablePipe](/docs/reference/debounce-frame-observable-pipe/)
  - [debounceImmediateObservablePipe](/docs/reference/debounce-immediate-observable-pipe/)
  - [debounceMicrotaskObservablePipe](/docs/reference/debounce-microtask-observable-pipe/)

- debug an Observable: [debugObservablePipe](/docs/reference/debug-observable-pipe/)


*with notifications*

- awaits to receive a `complete` or `error` Notification, and performs some kind of `mergeMap`:
  - **[thenObservablePipe](/docs/reference/then-observable-pipe/)**
  - [fulfilledObservablePipe](/docs/reference/fulfilled-observable-pipe/)
  - [rejectedObservablePipe](/docs/reference/rejected-observable-pipe/)
  - [finallyObservablePipe](/docs/reference/finally-observable-pipe/)
  - [thenAnyObservablePipe](/docs/reference/then-any-observable-pipe/)

- converts an Observable of Notification to an Observable:
  - accumulates `next` values in an array and send then when a `complete` is received:
  [notificationsToValuesObservablePipe](/docs/reference/notifications-to-values-observable-pipe/)
  - stores the last `next` value and send it when a `complete` is received:
  [notificationsToLastValueObservablePipe](/docs/reference/notifications-to-last-value-observable-pipe/)
